1. fbr24.getURL() возвращает url запроса. URL подставляется, меняем только в одном месте в конфиге.

2. $.extend(settings, options) соединяет оба объекта в один первый - settings.

3. $.param(query) создает сериализованное представление объекта query (т.е. создает строку).

4. $.ajaxSetup - предустановки для аякс запроса. Как кидать, что будет приходить в ответ и т.д.

5. Pnotify - подключенный плагин для оповещения в модальных окнах.

6. e.target - элемент, на котором произошло событие.

7. .data('.selector') - возвращает данные, установленные в selector.

8. fbr24.modal() - метод для открытия-закрытия модальных окон.

9. $.magnificPopup.open() - метод для открытия модального окна. В качестве аргументов принимает объект.

10. $.magnificPopup.close() закрывает модальное окно.

11. fbr24.addLoader() - метод, который отображает прелоадер.

12. fbr24.removeLoader() скрывает прелоадер.

13. String.prototype.replaceAll() заменяет все в строке.

14. $.fn.serializeObject() сериализует форму.

15. serializeArray() кодирует элементы формы в массив объектов из имен и значений.

16. fbr24.generateRandomID() рандомный ID заданной длины.

17. fbr24.dateDiff() разница между двумя датами.

18. fbr24.numberFormat() форматирует числа для денежных единиц.

19. fbr24.numberToText() преобразование числа в текст ( 1 = 'один').

20. fbr24.getPercent() возвращает процент от числа.

21. fbr24.utoa() кодирует в BASE64.

22. fbr24.atou() раскодирует из BASE64.

23. fbr24.getClientShortName() возвращает короткое имя для ЮЛ типа ООО, или ИП.

24. fbr24.createTab() создает tab.

25. fbr24.decodeHTML() преобразовывает спецсимволы в HTML сущности.

26. fbr24.getNumEnding() склоняет числительные.

27. fbr24.isCorrectDate() проверяет дату на корректность.

28. fbr24.initValidationForm() инициализирует валидацию форм.

//По видео Все запросы кидаются в JSON формате строкой. Отправляется подзапрос строкой, которая представляет из себя JSON. dataFilter - глобальная штука, в которой идет распарсивание того, что прислал сервак, он изначательно через try/catch пытается распарсить ответ серваера. Если это строка, то он пытается еще раз распарсить. Все кидается в консоль. Если нет определенных полей в ответе сервера, то светится сообщение через PNotify. С сервака может прийти несколько статусов, либо это будет error, либо еще что-то. В JSRender можно писать js-код через *. Через ~ можно вызывать вспомогательные методы (берется из функций для шаблонизатора).
